## 2026-07-21 · v10.3（姿态语义绑定：避免“只换表情”的过度修正）

v10.2 为阻止动作图误复用，把对话差分限制成同一中性姿势，导致生图 prompt 只允许表情变化。现改为三层表达：

- `dialogue_pose`：表情与身体语言一起变化，可使用抱臂、叉腰、插兜、前倾、扶额、手扶胸口等不会制造新剧情事实的手势
- `action`：递出剧情道具、止血、牵手、离场、奔跑等只允许绑定指定 visual beat
- CG：承担复杂双人交互和空间关系

每张 dialogue pose 必须声明 `gesture + usage_tags`；剧本每次切换时写 `; @pose:<tag>`，SCRIPT 校验器核对标签与资产用途。由此保留丰富姿态，同时避免“担心台词配递物图”“通讯离场仍挂正面图”。

## 2026-07-21 · v10.2（视觉节拍契约：从生成源头解决图文错位）

全路线实玩发现：文字写“熄灯/红光/人物已离场/只剩通讯杂音”，画面仍保留亮背景和正面立绘；同时带递物、跪地动作的立绘被当作普通“温柔/担心”表情反复复用。问题根因不在截图覆盖率，而在 SCRIPT 与 ASSETS 没有共享视觉语义。

- SCRIPT 新增强制 `visual_plan.json`：人物在/离场、灯光状态、具体肢体动作和道具状态必须先建 beat，再写 WebGAL 文本；视觉命令必须出现在叙述文字之前
- 新增 `validate_visual_plan.py` 作为 SCRIPT Gate，检查 `; @visual:<id>` 绑定、环境/动作/离场承载方式和命令顺序
- ASSETS 将 `expression_idle` 与 `action` 分离：idle 双手空、姿态中性；动作立绘写明 `allowed_beat_ids`，不得跨场景当情绪差分复用
- `check_assets.py` 在 ASSETS Gate 对照 visual plan 与 manifest，确保生图没有篡改 pose/action/props 语义
- INTAKE 的资产预算改为“中性表情差分 + 专用动作/CG + 背景状态变体”，不再用固定 6 张立绘硬扛所有剧情动作

用户验收发现"23 标题/HUD/动效都是重复的，21 的 HUD 动效也是重复的"。

   title_particles.png`（白色光斑）随装配存活——4 个项目都没自产 tile，

## 2026-07-18 · 验收蒸馏版（基于 4 case 实战验收）

本轮改动全部来自 5 个测试 case（4 个实跑：《雨停之前》《同一场雨》《第三小时》×2）的

### 新增

- **`scripts/assemble_build.sh`** — 一键装配：清 build → 引擎基座 → 内容覆盖 →
  主题 → config 补丁 → 五项自检（语言补丁/占位符/封面/模板/config），失败非零退出

  type:color:count，`apply_theme.sh` 据此判断重生成；`--keep-particles` 保留自定义 tile
