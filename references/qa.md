# QA：验收标准与回炉规则

## 验收清单（全部通过才算 QA 过）

1. **剧本**：`validate_script.py` 零错误
2. **资产**：`check_assets.py` 通过（引用存在、立绘带透明通道）
3. **E2E**：`scripts/e2e_test.py <build的本地URL> --out qa_report/ --scene-dir <场景目录> --max-steps <章数×25>`（**--scene-dir 必传**：文本感知点选选项，不传则退化为几何探测会误选；步数按章数×25，5 章≈130 步、每步约 7s，用 nohup 后台跑）全程无 JS 报错，且：
   - 标题页正常渲染（主题皮肤生效）：直达标题页无语言选择、标题文字已烘焙进背景、只有「开始游戏」单按钮、背景带 Ken Burns 循环动画、右上角无白色残留块
   - 对话可推进到至少一个 `end;`
   - 至少触发一次选择支并点击成功
   - 游戏内底部无控制条按钮
   - 场景切换/立绘变化带过场（剧本 -enter/-exit 参数生效，画面不是瞬切）
   - 重伤分歧点出现「重新来过」回溯选项，且回溯后不会无限循环
   - 情绪节拍处立绘差分有对应反应（训斥→得意、被抓包→惊讶），不是一张 normal 挂到底
   - 换背景/CG 时立绘先退场再换景，角色不"贴"在新场景上
   - 立绘正脸可见、无头发遮脸、无水印残留
4. **视觉检查**（逐张看 qa_report/ 截图）：
   - 文字不溢出对话框、不被裁切
   - 立绘完整显示、无白边绿边残留、位置居中
   - 背景/CG 是竖构图，无横向拉伸变形
   - 皮肤配色与 GDD 主色一致
   - **名牌约定**：角色（含主角）说出口的台词均带名牌且名字与 GDD 主角名牌名逐字一致；无名牌旁白正文可辨弱化（不透明度约 0.8），旁白/对话交替时文本框布局无跳动；对话框无与题材无关的装饰图形
5. **好感度逻辑**：用 `--choice-strategy first` 和 `--choice-strategy last` 各跑一遍 E2E，两条路线应到达**不同结局**，且与 GDD 结局设计对应（first=全选第一个选项通常好感最高，last=最低）
6. **多角色路线覆盖**（多攻略模式）：GDD 结局映射表里的**每个结局都要有一条可达路线**——除 first/last 外，用「定向偏好策略」补跑（如全程偏向角色A的选项 → 应达 A 单人结局或 HE）；某个结局不可达 = SCRIPT 数值阈值设计错误，回炉
7. **主题适配验收**：HUD 配色/对话框材质/选择支样式、封面氛围、粒子意象与配色，必须与 GDD「主题配方」节一致；且 GDD 里每个主题参数**必须带推导依据句**（从角色/剧情意象出发），出现无依据的参数值或明显的题材查表痕迹（如悬疑一律雨夜蓝）= INTAKE 推导缺失，回炉；粒子 tile 文件名 `background/title_particles.png`；动效为项目 `effects.css` 注入（index.html 与 userStyleSheet.css 均无 `__FX_CSS__` 残留），effects.css 头部有推导句，且两帧截图（间隔 ≥2s）粒子位置可辨变化
8. **返回标题页复检**（结局后路径）：`end;` 回到标题页后**停留 2s 再截图判读**——封面层 remount 有约 0.5-1.5s 解码瞬态（build.md 第 5 节），瞬态内的备份背景色块不算缺陷；2s 后仍有残留色块才算 BUILD 缺陷回炉。同时确认封面已瘦身（1440 宽 progressive，≤500KB），且**封面标题文字逐字校对无错字**（直出封面必经步骤，错字回 ASSETS 重抽或降级 bake_title）

## 回炉规则

| 失败现象 | 回到阶段 | 动作 |
|---|---|---|
| 结局不可达/标签缺失 | SCRIPT | 修剧本，重跑 validate_script |
| 立绘不像原图/崩坏 | ASSETS | 按 art-director.md 换 seed/prompt 重生成该张 |
| 资产缺失/无透明通道 | ASSETS | 补生成或重抠图 |
| 文字溢出/样式异常 | BUILD（皮肤） | 改主题 SCSS，重新 apply_theme |
| 对话框/选择支整个不渲染 | BUILD（皮肤） | 主题 SCSS 缺结构类——以 anime 完整版为骨架逐类补齐（见 build.md 主题配方节骨架铁律） |
| 双人同屏立绘归中叠图 | SCRIPT | 引擎站位渲染归中，改单人切换写法（scriptwriter.md 多角色演出节） |
| 多攻略某结局不可达 | SCRIPT | 重算数值阈值（五步法第 4-5 步），补可达路线 |
| 点击无响应/JS 报错 | BUILD | 查控制台报错，修引擎或剧本 |
| 结局与 GDD 不符 | INTAKE（严重时） | 与用户重新对齐 GDD |

## 已知环境注意事项

- 无头浏览器里 `text=xxx` 的可见性判定在 transform scale 容器下不可靠；**点击一律用「div 文本定位 + bounding box 坐标点击」**（e2e_test.py 已实现此方法）
- **404 白名单**：`/api/webgalsync` WebSocket 报错是引擎联机同步功能在单机版的固有噪音，不算失败；其他 404（字体/图片/剧本文件）必须查清
- 下载/拷贝来的资产入库前先 `file` 验证是真图片（下载管道会把 JSON 错误页存成 .png 文件名）
- macOS 环境注意：skill 内脚本一律用 perl 做就地替换，手写一次性命令时别把 GNU `sed -i` 带进来（BSD sed 会报 undefined label）
- iOS Safari 真机未覆盖时，交付说明里标注「建议真机复测」
- 交付前提醒：存档页/设置页为引擎默认样式，属已知简化项
